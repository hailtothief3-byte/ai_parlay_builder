# Feature Map

## Product framing

AI Parlay Builder is best understood as a prop workflow and decision-support platform with four primary jobs:

1. `Scan`
2. `Build`
3. `Track`
4. `Learn`

## Core workflow surfaces

### Overview

- current posture
- next-step coaching
- top priority strip
- workflow guidance

### Live Board

- normalized live board inspection
- quick filters
- export-friendly table views

### Edge Scanner

- ranked prop candidates
- smart ranking support
- market and book filtering

### Parlay Lab

- sportsbook build workflow
- demo workflow
- DFS auto-slip workflow
- smart parlay profile support

### Results & Grading

- tracked picks
- settlement queue
- saved tickets
- bankroll journal
- ticket review

## Intelligence layers

### Smart Pick Engine

- history-informed scoring
- recent-form memory
- auto-tuning
- manual override support
- audit breakdowns

### Smart Parlay Profiles

- recommended leg count
- confidence posture
- same-player / same-team guidance
- DFS destination preference memory

### Review system

- weekly review
- monthly review
- recommendation cards
- coach mode
- action checklist

## DFS support

Current DFS destination support includes:

- PrizePicks
- Underdog Fantasy
- Chalkboard
- Betr Picks
- ParlayPlay
- Dabble
- DraftKings Pick 6

Current product stance:

- generate and package slips inside AI Parlay Builder
- support handoff/open workflows
- do not promise universal public external prefill or auto-submit

## Packaging layers

### Simple view

Best for:

- demos
- buyers
- calmer day-to-day use

Hides heavier diagnostics by default.

### Pro view

Best for:

- deep review
- tuning
- experimentation
- smart-engine inspection

### Product plans

- `Core`
- `Pro`
- `Owner`

These currently act as packaging posture controls and provide a natural basis for monetization or handoff positioning.
